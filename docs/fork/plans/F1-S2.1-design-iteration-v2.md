# F1-S2.1 — Design iteration v2 (responsive collapse · shade/contrast system · motion)

Slice plan per CLAUDE.md § Iteration. Triggered by the maintainer's review of the live F1-S2
cockpit (2026-06-12, verbatim in HANDOFF § Next slice item 1). Linked ADRs: F006 (design
system — this slice EXTENDS its token layer, no new ADR needed), F002 (glass cockpit).
Standing rule, verbatim: "don't use black background, needs to be clean and professional,
cutting edge design."

## Benchmark review (screenshots this time, as instructed)

Method: product screenshots downloaded from the four benchmark products' own sites and read
visually (Harvey assistant UI 2740×1540 from harvey.ai/products/assistant; Legora composer /
citation card / split workspace from legora.com/product; Linear changelog product shots +
dark cards; Attio og canvas), reviewed against full- and half-width captures of our live
cockpit ("before" shots in `docs/fork/evidence/f1-s21/`). Benchmark images are referenced by
URL and kept out of the repo (third-party content).

Converging idioms across all four:

1. **Surface separation is decisive.** Legora floats white rounded panels with soft
   diffused shadows on a clearly darker warm-gray canvas (≈6–7 L% delta). Harvey inverts it
   (near-white canvas, composer FILLED slightly darker + border). Linear dark: canvas ≈L 0.14
   under cards ≈L 0.23 with hairline internal dividers. Our current canvas 0.985 vs card 1.0
   is an invisible 1.5% — this is the root of "no shades, contrasts don't make sense".
2. **One focal surface per view**; chrome stays quiet at the edges (Harvey's composer is the
   page; matter picker and library link are small gray text above it).
3. **Hierarchy is stepped text + hairline section dividers inside cards** (Linear card:
   title → body → metadata in three clearly distinct strengths; 24px padding).
4. **Pills/chips for tools and status; soft shadows scale with elevation; focus rings are
   visible** (Legora's blue composer focus glow).
5. Empty/loading states are designed, not afterthoughts (Legora skeleton document lines).

Our before-state confirmed failures (screenshots): rail squashes to ~127px at 700px viewport
with text collisions; embedded legacy ConversationPanel overflows horizontally even at
1280px; three vertical regions read as one flat sheet split by hairlines; zero resting
shadows; motion vocabulary is two hover transitions; dark mode leaves the legacy panel
hard-white (no `.dark` overrides exist for `--lq-*`) and native scrollbars stay light
(no `color-scheme`). The "header stayed light in dark capture" was probed live and is a
Cypress paint-timing artifact, not a bug (computed styles flip correctly).

## Goals

1. **(a) Responsive collapse** — the cockpit works at half-screen width:
   - Wide (≥880px): rail pane becomes `collapsible` (paneforge 1.0.2 verified API:
     `collapsible`, `collapsedSize={0}`, `minSize`, `onCollapse`/`onExpand`, instance
     `collapse()`/`expand()`; collapsed state persists under `autoSaveId`; explicit `id` +
     `order` set on panes). Header gains a rail-toggle button (PanelLeft icon).
   - Narrow (<880px viewport): the rail leaves the pane group entirely and becomes an
     off-canvas drawer (fixed, w-72, fly-in + scrim `bg-foreground/20`, Escape/scrim/select
     closes). No new dependency — ~30 lines, bits-ui not needed.
   - ConversationHost: at <720px container width the fixed `w-72` thread aside stacks —
     list full-width OR conversation full-width with a back row.
   - Fix the legacy panel's horizontal overflow inside the cockpit (visible at 1280!).
2. **(b) Shade/contrast/motion system**:
   - Token layer v2 in `app.css` (extends ADR-F006 vocabulary, same names):
     light canvas drops to `oklch(0.965 0.004 90)` (decisive warm gray under white cards),
     foreground crisper (0.21), muted-foreground darker (0.45) for a real 3-step text
     hierarchy, border 0.90; sidebar tokens align to canvas (rail merges with canvas,
     content floats — the Legora idiom). Dark: card lift +0.05 over canvas (0.28 on 0.23
     floor — never black), `color-scheme: dark` so native scrollbars/controls flip.
   - **Elevation scale (new vocabulary)**: `--elevation-xs/sm/md/lg` custom props
     (light: layered low-alpha cool-tinted shadows; dark: deeper alphas) mapped in `@theme`
     to `--shadow-xs/sm/md/lg` so `shadow-*` utilities are tokenized. Resting `shadow-xs`
     on item cards, `shadow-sm` on the workspace card, `md` popovers, `lg` dialogs.
   - Cockpit restyle on the new scale: ConversationHost becomes an inset rounded-xl
     workspace card (white, shadow-sm) with a recessed `bg-muted/40` thread aside; area
     cards get resting shadow + hover lift + arrow affordance; matters rows get hover
     chevron; selected thread rows get `text-accent-foreground` (missing today); resizer
     gets hover/active affordance; skeletons match real row sizes; empty states get icon
     circles; StatusPill unchanged (already the intent-token pattern).
   - Motion: `in:fade` view transitions (120ms, reduced-motion aware via a helpers.ts
     gate), `scroll-smooth` + `overscroll-contain` on the four scroll containers,
     standardized 150ms ease-out hover transitions, pane collapse animates via
     `transition-[flex-grow]` (paneforge `data-pane-state` contract).
   - **Legacy dark stopgap (wave 0)**: `.dark { --lq-* }` override block in `practice.css`
     mapping the 30 legacy tokens onto the charcoal scale — fixes dark mode for every legacy
     surface in one file; per-surface migration supersedes later.
   - **ConversationPanel shell harmonization ONLY** (it renders inside the cockpit and
     leaks the old design system into the flagship surface): canvas/card/border/Run-button/
     label tokens in its scoped CSS map to semantic tokens (`--background`, `--card`,
     `--border`, `--primary`, `--muted-foreground`); the full 113-use migration of its
     message tree stays in rollout wave 1.
3. **(c) Whole-interface rollout — PLANNED here, executed in later slices** (per the
   maintainer: "once done, the entire interface will need to change, not just the agents
   tab"). Order (from the legacy inventory, 71 components, ~1,095 `--lq-*` uses):
   - **Wave 1 — conversation core**: full `agents/ConversationPanel` (1,412 LOC) then the
     `ChatPanel` tree (MessageList/MessageBubble/ChatSidebar/composer satellites) —
     restyles chats, matters workspace, agents tab AND the cockpit pane in one wave.
   - **Wave 2 — library surfaces**: knowledge, skills (+SkillWizard), playbooks(+executions),
     tabular. Mostly mechanical token swaps.
   - **Wave 3 — long tail**: settings, admin, trust, saved-prompts, learn; `autonomous/`
     only if it survives F2/F3 (slated for deep-agents replacement — don't gold-plate);
     `(tools)` chrome last (F3 demotes tool tabs anyway).

## Non-goals

- No legacy surface migration beyond the ConversationPanel shell + the one-file dark
  stopgap (waves 1–3 are separate slices).
- No login/change-password restyle (pre-token page, wave 3).
- No new vendored components, no dependency changes, no API/DB changes (web-only slice).
- No `@custom-variant dark` (darkMode comes from `@config tailwind.config.js` — gotcha on
  record); vendored `components/ui/**` files are not edited (stay diffable vs upstream).
- Token NAMES don't change — values and additions only; legacy `--lq-*` light values
  untouched.

## Deviations

- **ConversationPanel: full style-block token migration, not "shell only".** The plan
  scoped canvas/card/border/Run-button/labels; in practice the panel's entire legacy
  palette lives in ONE scoped `<style>` block, so migrating all of it (badges → status
  intent pairs, steps, insets, shimmer, ghost buttons) was the smaller, safer diff than
  splitting the block into migrated/unmigrated halves. Markup and `--lq-space-*`/pill
  constants untouched; rollout wave 1 now covers the panel's markup/typography plus the
  ChatPanel tree.
- **Centered reading column for the embedded panel.** Inside the cockpit the panel
  rendered edge-to-edge (it was built for the legacy page container); the host now wraps
  it in `max-w-3xl mx-auto` (the Harvey center-column) — host-level, no panel change.
- **`overflow-x: clip` on `html`** — the before-captures showed a body-level horizontal
  scrollbar even at 1280px; the document never legitimately scrolls (the shell owns
  scrolling), so the guard is unconditional rather than chasing each inner min-width.
- **Dark canvas probe**: the "header stayed light in dark mode" before-capture was
  disproven live (computed styles flip correctly — Cypress paint-timing artifact); the
  dark capture in the regression spec now settles 400ms before shooting.
