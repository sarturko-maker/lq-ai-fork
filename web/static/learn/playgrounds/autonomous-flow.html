The guarded_tool_call chokepoint — four ordered checks R5 → R6 → R4 → open span + audit + run + record
There is exactly one chokepoint. Every tool call goes through here; a new tool gets the brakes for free and cannot route around them. The order is fixed: R5 (halt) → R6 (phase grant) → R4 (cost) — all before the tool runs.
