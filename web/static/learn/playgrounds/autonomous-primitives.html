Autonomous primitives: watches · schedules · memory · precedent

The four building blocks of the autonomous layer. Each has its own trigger → session → outcome lifecycle. Pick a primitive, then click a step to see the real endpoints, tables, and state transitions behind it. (For what happens inside a single session — its phases and brakes — see the autonomous-flow playground.)

The whole layer is per-user opt-in, off by default (User.autonomous_enabled). Every state transition writes an audit row carrying counts, types, IDs, and enum values only — never raw document text or proposed values.
