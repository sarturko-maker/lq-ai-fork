What is a skill? A skill is an open-source, structured prompt artifact — a SKILL.md file containing a YAML frontmatter block and a plain-markdown body. The body becomes the system-prompt section injected into the chat when a user attaches the skill. Skills live in skills/ in the repository and are visible, forkable, and auditable by any user.

For attorneys: Write the body in plain English, as if briefing a careful paralegal — no programming required. The frontmatter fields (slug, title, description, version, tags) are mechanical labels; the only creative work is the body. The slash_alias field lets users invoke your skill by typing /your-alias in the chat input.

Attestation step: Per skills/CONTRIBUTING.md, every new skill PR requires a practicing-attorney attestation paragraph in the PR description — the human contributor is the attesting party. The skill must reflect that attorney's jurisdiction and practice area expertise.

Fork affordance: Any logged-in user can fork a built-in skill from the Skill Library, edit the copy, and save it as their own user-scope skill. The fork is independent — changes to the built-in after forking do not propagate. See skill-acceptance-tests.md for the per-skill test corpus contribution path.
